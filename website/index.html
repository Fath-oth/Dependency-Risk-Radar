End-to-End Pipeline

From source to risk report
in under 60 seconds

Four decoupled layers process Gradle projects and compiled APKs through a fully automated, real-time pipeline.

1

Ingestion

Accepts a Gradle project directory or compiled APK. The parser handles Groovy DSL, Kotlin DSL, and version catalogs, extracting direct and transitive dependencies by invoking gradlew :app:dependencies. APKs are analyzed with androguard.

2

Enrichment

Five external data sources are queried in parallel: OSV.dev, NVD, Maven Central, Exodus Privacy, and ClearlyDefined. Results are cached in Redis for 24 hours, reducing latency for repeated queries.

3

Scoring & Graph Analysis

A weighted formula computes a global risk score [0–100] for each component. A networkx DAG propagates risk transitively with exponential depth decay and identifies critical paths and high-impact nodes.

4

SBOM Generation & AI Planning

CycloneDX and SPDX SBOMs are generated with full metadata. The top-30 highest-risk components are sent to the Gemini API for a prioritized update plan. A PDF report is also produced for audit compliance.

5

Dashboard & Export

Results are served through a REST/WebSocket API to a React dashboard with D3.js graph visualization, filterable component explorer, AI update plan view, SBOM explorer, and historical score tracking.

Risk Scoring

Transparent, weighted
multi-criteria scoring

Every dependency receives a global risk score [0–100] computed as a weighted average of four independent sub-scores, all normalised to the same scale.

Scoring Formula

Risk = 0.45·SCVE + 0.25·Sobs + 0.20·Slic + 0.10·Strack

Transitive decay: indirect vulnerabilities are weighted as score × 0.8depth−1 — a CVE one hop away counts more than five hops away.

Architecture

Four decoupled layers,
one cohesive platform

DRR is structured into four independent layers communicating through well-defined interfaces, making each component independently testable and replaceable.

Layer 1 — Ingestion

Gradle Parsing & APK Analysis

Handles Groovy DSL, Kotlin DSL, and version catalogs. Invokes Gradle to obtain the full transitive dependency tree. Analyzes compiled APKs with androguard for class-level package inference and permission extraction.

Python 3.11androguardGradle CLIRegex DSL parser
Layer 2 — Analysis

Multi-criteria Scoring & Graph

Computes the four-dimensional risk score for each component. Builds a networkx DAG, detects high-impact nodes by in-degree, finds critical risk paths with nx.shortest_path, and aggregates transitive scores.

networkxSQLite / PostgreSQLSQLModelPydantic
Layer 3 — AI

Update Planning & Narration

Sends the top-30 highest-risk components to the Gemini Flash API with a structured system prompt. Generates per-component migration plans with priority, effort, and breaking-change risk. Falls back to rule-based planning if the API is unavailable.

Gemini Flash APIStructured promptsRule-based fallback
Layer 4 — Presentation

Dashboard, API & Exports

FastAPI backend with REST + WebSocket (real-time SSE progress). React 18 frontend with D3.js force-directed graph, Recharts dashboards, and shadcn/ui. Exports: CycloneDX, SPDX, PDF audit report, CSV.
